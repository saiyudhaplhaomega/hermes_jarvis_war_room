What is Context Isolation?

Context isolation ensures that subagents operate with their own separate context windows, preventing context bloat and improving performance. When enabled, each subagent's work (e.g., tool calls, intermediate results) is isolated from the main agent, reducing the risk of "context rot" and improving task completion rates.

When to Use Context Isolation

Enable for:

  • • Tasks requiring multiple steps (e.g., codebase exploration).
  • • Specialized subagents (e.g., Researcher vs. Writer).
  • • Multi-model workflows (e.g., small model for low-latency tasks).
  • • Parallel execution (e.g., multiple subagents running simultaneously).

Disable for:

  • • Simple, single-step tasks.
  • • Workflows requiring shared context (e.g., collaborative editing).
