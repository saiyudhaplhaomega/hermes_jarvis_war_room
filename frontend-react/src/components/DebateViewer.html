Should Hermes Jarvis adopt role-based agents?

Agents: Researcher-001, Engineer-002, Manager-003 Status: Active
R
Role-based agents improve clarity and scalability by assigning explicit roles (Researcher, Writer, Manager) instead of a monolithic prompt. This aligns with CrewAI's approach and reduces context bloat.
E
While role-based agents are useful, they add complexity to the codebase. We'd need to refactor the agent creation API, update the UI, and ensure backward compatibility. Is the ROI worth it?
M
The ROI is clear: role-based agents enable better task decomposition and collaboration. For example, a Researcher can gather data, a Writer can draft content, and a Manager can coordinate. This mirrors human teams and improves efficiency.
R
Additionally, role-based agents allow for specialization. A Researcher can use tools like Google Search and Web Scraper, while a Writer can use Grammarly and Notion. This reduces the cognitive load on each agent.
E
I agree with the benefits, but we should prototype this first. Let's create a minimal implementation with 3 roles (Researcher, Writer, Manager) and test it in a controlled environment.
